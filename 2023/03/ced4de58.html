Java12～17 新特

发表于 2023-03-07 更新于 2023-11-15 分类于 Java新特性 阅读次数： 本文字数： 2.2k 阅读时长 ≈ 8 分钟

ps: Java 的新特性有一个预览版本，预览版本的新特性会在两个版本后更新到正式版，编译执行代码预览版本的特性的 Java 文件需要添加 --enable-preview 参数

本篇所有内容摘自：https://pdai.tech/md/java/java8up/java12.html

Java12

新的 switch 语句支持（preview）

  • case 项支持合并在一起
  • case 的执行语句支持使用 lambda 的形式

Shenandoah 低暂停时间垃圾回收器（实验）

JDK 12 引入的回收算法（实验阶段），该算法通过与正在运行的 Java 线程同时进行疏散工作来减少 GC 暂停时间。Shenandoah 的暂停时间与堆大小无关，无论堆栈是 200 MB 还是 200 GB，都具有相同的一致暂停时间。

ps: Shenandoah 只存在于 OpenJDK 中，可以使用 -XX:+UseShenandoahGC 命令行参数打开

Shenandoah 适用于高吞吐和大内存场景，不适合高实时性场景 。Shenandoah 算法设计目标主要是响应性和一致可控的短暂停顿，对于垃圾回收生命周期中安全点停顿（TTSP) 和内存增长监控的时间开销并无帮助。

Shenandoah 算法为每个 Java 对象添加了一个间接指针，使得 GC 线程能够在 Java 线程运行时压缩堆。标记和压缩是同时执行的，因此我们只需要暂停 Java 线程在一致可控的时间内扫描线程堆栈以查找和更新对象图的根。

Java13

switch 支持返回结果（preview）

  • 上述两种 switch 的写法都不需要添加 break 关键字来中断 case，所以也不会出现忘写 break 而出现 switch 穿透的问题
  • 如果是使用 case 后面添加代码块的方式，需要使用 yield 关键字返回当前 case 块需要返回的数据
    • ps：yield：产出、提供 的意思

文本块（preview）

  • 文本块可以省去我们手动添加 \n 和拼接字符串的烦恼，可以联系 js 中使用 `` 包裹的字符串，除了不能使用 ${} 来直接替换变量，其他的也差不多

Java14

switch 特性提升为正式版

switch 表达式的新特性使得 switch 语句在编码方式上简化不少，但是同时还是需要注意：

  • 需要保持与之前 switch 语句同样的 case 分支情况。
  • switch 表达式中不再需要显式地使用 return、break 或者 continue 来跳出当前分支。

NPE 提示增强，可以提示到是哪个变量引起的 NPE

使用 -XX:+ShowCodeDetailsInExceptionMessages 开启

instanceof 模式匹配语法增强（preview）

  • instanceof 后面可以直接声明的变量，声明的变量是判断的类型，作用范围是之后的 if 块
  • 在 && 运算符中，&& 前面使用 instanceof 声明的变量可以在 && 之后使用，即：变量的作用范围延长了，但是这种延长不适用于 || 运算符，因为 || 运算符即使前面没有匹配成功，也会执行后面的判断，此时使用会产生问题

Record 类型（preview）

Record 类型允许在代码中使用紧凑的语法形式来声明类，而这些类能够作为不可变数据类型的封装持有者。Record 这一特性主要用在特定领域的类上；与枚举类型一样，Record 类型是一种受限形式的类型，主要用于存储、保存数据，并且没有其它额外自定义行为的场景下。

record 类型的实体反编译后会继承于 java.lang.Record 类【与枚举类型类似】，且拥有部分特征：

  • 有且仅有一个构造方法
  • 拥有获取成员属性的方法：name()
  • 重写了 hashcode() 和 equals()
  • 重写了 toString()
  • 类对象和属性均被 final 修饰，表示类不可被继承，属性不可被再次赋值

同时，Class 中也引入了新的方法

文本块增强（preview）

引入了 \ 和 \s

  • \ 与各种配置文件中的用法一致，表示当前行没有结束
  • \s 表示一行末尾的空格，因为文本块会删掉一行末尾的 \r\n，同时也会删除末尾的空格
    • 注意：代码第 12 行的空格并不会在控制台打印出来

接下来的特性对于使用 Java 进行开发的开发人员关系不大

打包工具（孵化版）

用于将 Java 项目打包成可在 windows、macos、linux 系统直接运行的程序包

在 Java 13 发布前的最后一刻被从外围移除的打包工具 jpackage 终于可以在 Java 14 中使用了。但是要注意，它只在 Incubator 版本中提供。

这个打包工具应该可以让开发人员创建自主的 Java 应用程序，它以 JavaFXjavapackager 这个打包工具为基础。它的主要特点有如下三个：

  • 支持本地打包格式，为用户提供自然的安装体验；
  • 打包时可以指定启动参数；
  • 通过命令行或使用 ToolProvider API 的程序来启动。

某个组合垃圾回收器被弃用

Java 14 打算弃用 ParallelScavenge + SerialOld 垃圾收集器组合。

支持这一改变的 JEP 366 清楚地表明，他们的目的不是要删除掉这个组合，而是要弃用它。

弃用这种算法组合的原因是：它很少被使用，同时需要大量的维护工作。

Java15

密封的类和接口（preview）

密封的类和接口用于显示那些类可以继承当前类，限制了类的层次结构，是一个很实用的特性

Java16

instanceof 模式匹配增强提升为正式版

Record 类型提升为正式版

向量 API（孵化版）

向量 API 存在于 jdk.incubator.vector 包中，属于 jdk.incubator.vector 模块。向量计算也成为矢量计算，指的是充分运用多核 CPU 的特性，使多个计算并行进行，同时计算

密封类提升为正式版

关于参数，建议看 javadoc 或注释

  • 这一大坨就是可选的参数

Java19 开放了预览版的 虚拟线程，结构化并发也已进入孵化阶段，推测这两块新的 API 将是很大的亮点
