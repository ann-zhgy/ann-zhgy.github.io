ES 入门

安装

Java 安装

  • elasticsearch 需要 Java 的环境，我们一般需要安装 JDK1.8 或以上版本，Java 的安装步骤不再展示（参考菜鸟教程）……

Elasticsearch 安装

官网下载安装包安装

  1. 访问官网下载安装包

    可以在 官网 直接下载，也可以选择 Elasticsearch 和 Kibana 版本下载，也可以在官方 github 下载源码

Kibana 是 Elastic 公司提供的免费且开放的用户界面，方便用户对 Elastic 进行各种操作

  1. 解压之后，在 bin 目录运行 elasticsearch.bat 即可

  2. 运行完成之后，访问 127.0.0.1:9200，出现如下界面表示运行成功

    <img alt="20210614205247" src="https://i.loli.net/2021/06/14/oyxaB4VSnRrIhcO.png" style="width: 100%;">

  3. ES 目录介绍（需要我们关注的有四个）

    目录 作用
    bin 存放了启动及设置 ES 的入口（可执行文件）
    config 存放了 ES 的配置的信息
    data 存放数据
    logs 存放了 ES 运行过程中产生的日志信息
    plugins ES 支持的各种插件存放在该目录中

docker-compose 安装

  1. 准备配置文件

    • elasticsearch.yml：可以参考下载下来的包中的 config 中的 elasticsearch.yml 文件
    • kibana.yml：同理，参考下载下来的包中的 config 中的 kibana.yml 文件

  2. 编写 docker-compose

  3. 启动，访问 http://ip:port 验证是否安装成功

    1. 启动前需要将配置文件复制到相对应的挂载目录 (volumes) 中，然后才可以成功启动容器
    2. 这个 docker-compose 是一个单一节点的案例

ik 分词器下载安装

从 github 选择相对应的版本下载，然后放在目录 plugins/ik 下面，重启 ES 就好了

Elasticsearch 配置文件解析

ES 的配置文件在 config 文件夹下，值得关注的有 elasticsearch.yml、jvm.options、log4j2.properties，但是我们需要关注的仅仅是 elasticsearch.yml，jvm.options 在 JVM 的笔记中会详细解释，log4j2.properties 里面存储了日志打印的信息，一般不用管，而且在 elasticsearch.yml 中同样可以进行日志的配置

elasticsearch.yml

  • 更多配置请访问官方文档

jvm.options

对于学习来说，这个配置文件中我们只需要关注两个参数即可

  • 这两个参数用于设置 JVM 的内存，两个值设置为一致大小可以防止内存抖动（一个很正常的现象），ES 学习时推荐设置 512M 就基本足够了

ES 的数据结构设计

逻辑设计

包含文档、类型、索引，可以理解为分别对应关系型数据库的 记录（行）、表（更像是表中的列的描述）、数据库，但是区别在于：
关系型数据库中一个数据库中可以有多张表，但是 ES 从 6 开始，建议一个索引中只能有一个类型，在未来版本中会规定一个索引只有一个类型

文档（document）

elasticsearch 是面向文档的，意味着它搜索和索引数据的最小单位就是文档。文档有几个属性：

  • 文档是 key-value 的形式，需要同时有值
  • 文档中可以包含另一个文档，可以理解为 json 的 value 也可以是一个 json
  • 结构灵活。文档中，字段不需要预先定义，可以忽略某个字段，甚至可以可以动态的添加字段
  • 无模式。文档的字段对应的值可以是任意类型
    • 有一点需要注意，尽管文档是无模式的，但是字段的类型依旧很重要，明确的类型可以提高存储和索引时的效率

类型（type）

类型规定了文档的字段的类型

类型中对于字段的定义称为映射，比如将 name 映射为字符串类型，age 映射为整型

ES 中的文档是无模式的，当文档中的字段没有映射类型时，ES 会猜该字段是什么类型。但是 ES 猜的那个类型也不一定是我们想要的类型，所以比较好的办法的是提前定义好所需要的映射

索引

elasticsearch 的索引是一个非常大的文档集合，存储了文档和定义好的映射（如果有）

物理设计

节点和分片

es 是支持集群部署的，一个集群包含至少一个节点，一个节点就是一个 es 的实例（进程）。节点内可以有多个索引

一个索引默认有 5 个分片（primary shard），每个分片又有一个副本（replica shard）

当集群有三个节点时：

可以看到主分片和对应的副本分片不会分布在同一节点上，这样就算某个节点挂了，数据也不会丢失

实际上一个分片就是一个 Lucene 的索引，一个倒排索引的目录

倒排索引

索引可以理解为目录，一般是类似于 标题 —— 页码 的 key-value 结构。但是当我们使用关键词进行搜索时，就需要扫描索引库的所有文档，当文档很多时，这样的索引结构根本无法满足近实时的需求

倒排索引是将原本的索引结构转换为 关键词 —— 文档 ID 的映射结构，每个关键词都对应包含此关键词的文档的 id 列表

举个栗子

比如说，我们现在有两个文档，分别包含如下内容：

倒排索引创建后，是这样子的：

term doc1 doc2
It √ √
is √ √
never √ √
too √ √
late √ √
to √ √
mend √ ×
learn × √

现在，我们搜索 to learn，只需要搜索每个词条包含的文档就好了

term doc1 doc2
to √ √
learn × √
total 1 2

可以看出，两个文档都有匹配的地方，第 2 个文档匹配度更高
