JAVA8 新特性

发表于 2021-06-13 更新于 2023-11-15 分类于 Java新特性 阅读次数： 本文字数： 3.5k 阅读时长 ≈ 13 分钟

Java7 和 Java8 有很多新的不同，这里仅仅说 API 层面，其重点是 StreamAPI 和 Lambda 表达式，这两种写法可以让我们就像说话一样的编程

接口的默认方法和静态方法

默认方法

其实默认方法是 Java 更新中很无可奈何的扩展机制，因为对于大多数的接口来说，其已经有很多的实现，但是对于需要在接口上新添加的方法来说，不论是对于使用方还是 jdk 本身来说，其每个实现类都增加实现是一件工程量很大的事情，如果有默认实现可以减少很多的工程量。

  1. 正常使用

    Java 8 允许我们给接口添加一个非抽象的方法实现，只需要使用 default 关键字即可，这个特征又叫做扩展方法，示例如下：

  2. 两个接口中同名同参方法的覆盖

    如果一个接口中定义了一个默认方法，而另一个父类或接口中也定义了一个同名同参的方法（无法构成重载）

    • 类优先原则

      1. 选择父类中的方法。如果一个父类中提供了具体实现，那么接口中具有相同名称和参数的默认方法将会被忽略

      2. 接口冲突。如果一个父接口提供了一个默认方法，而另一个父接口也提供了一个具有相同名称和参数列表的方法（不论是否为默认方法），那么必须覆盖此方法来解决冲突

静态方法

Java8 中允许添加静态方法，但是静态方法必须提供实现。调用时使用静态类的调用方式即可

Lambda 表达式

Lambda 的本质是一个匿名内部类，我们可以把 Lambda 表达式理解为是一段可传递的代码（将代码像数据一样传递，就像 js 里面的 hock / 钩子函数），可以写出更简洁、更灵活的代码。作为一种紧凑的代码风格，它使 Java 语言的表达能力得到了提升。Java 中的 lambda 表达式可以理解为原来的匿名内部类的一个语法糖

lambda 表达式代替匿名内部类的必要条件

  1. 此匿名内部类的类型必须是接口
  2. 此接口必须有且仅有一个待实现的方法（可以使用 @FunctionalInterface 进行编译时检查）

lambda 表达式语法

  1. 基础语法

    Java8 引入了 箭头 / Lambda 操作符 (->)，它将 lambda 表达式分为左右两部分：

    • 左侧：lambda 表达式的参数列表
    • 右侧：lambda 表达式中所需执行的代码，即：lambda 体

  2. 特殊格式

    • 左侧的参数列表是有类型的，但是一般可以不用写，因为 Java 会自动做类型推断

    • 左侧没有参数时，() 不可以省略

    • 左侧只有一个参数时，() 可以省略

    • 右侧只有一行代码时，不论是方法需不需要返回值，{} 都可以省略

StreamAPI

Stream 是 Java8 中处理集合的关键抽象概念，它可以指定你希望对集合处理的操作，可以执行复杂的查找、过滤、和映射数据等操作。 使用 Stream 对集合数据进行操作，就类似于使用 SQL 执行的数据库查询。同时，它还提供了并行操作，简单的说： StreamAPI 提供了一种高效且易于使用的数据处理方式，尤其是对于大集合数据的处理性能很高

Stream 操作

  1. 创建 Stream 流：从一个数据源中获取流

  2. 中间操作（仅展示常用方法）：操作的中间链，对数据源的数据进行操作。每一次中间操作都会返回一个新的流

    stream 所有的中间操作都是需要接受一个 lambda 表达式（匿名内部类）的

    1. 筛选和切片

      • filter(Predicata p)：执行断言型接口，从流中过滤掉不符合条件元素
      • distinct()：去除流种重复的元素（需要元素实现 equals() 和 hashCode()）
      • limit(long maxSize)：截断流，使其元素不超过给定数量。多用于无限流
      • skip(long n)：跳过元素，返回扔掉前 n 个元素的流，若流中元素不足 n 个，则返回空流，与 limit (n) 互补

    2. 映射

      • map(Function f)：接受一个函数作为参数，该函数会被应用到每一个元素，并将其映射为一个新的元素
      • mapToDouble(ToDoubleFunction f)：接受一个函数作为参数，该函数会被应用到每一个元素，并产生一个新的 DoubleStream
      • mapToInt(ToIntStream f)：接受一个函数作为参数，该函数会被应用到每一个元素，并产生一个新的 IntStream
      • mapToLong(ToLongStream f)：接受一个函数作为参数，该函数会被应用到每一个元素，并产生一个新的 LongStream
      • flatMap(Function f)：接受一个函数作为参数，该函数会被应用到每一个元素使其转换为一个流，最后会将这些流连接为一个流

      运行结果

    3. 排序

      • sorted(Comparator<? super T> comparator)：按照指定的比较器排序

    4. 其他操作

      • peek(Consumer<? super T> action)：可以从流中获取元素对其进行操作，该方法主要是用来调试的

  3. 终止操作：执行中间链，并产生结果

    1. 查找和匹配

      • allMatch(Predicate<? super T> predicate)：判断流中所有元素是否与 lambda 匹配
      • anyMatch(Predicate<? super T> predicate)：判断流中是否存在元素与 lambda 匹配
      • noneMatch(Predicate<? super T> predicate)：判断流中所有元素是否不与 lambda 匹配
      • findFirst()：返回流中的第一个元素，无序流将随机返回元素
      • findAny()：随机返回流中的元素
      • count()：返回流中元素总数
      • max(Comparator<? super T> comparator)：返回流中元素的最大值
      • min(Comparator<? super T> comparator)：返回流中元素的最小值
      • forEach(Consumer<? super T> action)：遍历
      • forEachOrdered(Consumer<? super T> action)：顺序遍历

    2. 归约

      • reduce(T identity, BinaryOperator<T> accumulator)：遍历将流中元素并将其结合起来，得到一个值，返回 T
      • reduce(BinaryOperator<T> accumulator)：遍历流中元素并将其结合起来，得到一个值，返回 Optional<T>

    3. 聚合

      • collet(Collector c)：将流聚合为指定形式

        参数：

        • Collector.toList()：将流中元素收集到 List
        • Collector.toSet()：将流中元素收集到 Set
        • ......

      • Collector 中实现了很多聚合操作，还有 groupBy()、mapping() 等方法

Optional 类

Optional<T> 是一个容器类，代表一个值存在或不存在，可以和那程度上避免空指针异常（NPE），其主要应用于用于链式编程中的判空

常用方法

  • of(T value)：返回带有指定的当前非空值的 Optional。传入值为空抛出空指针异常。

  • ofNullable(T value)：返回一个描述指定值的 Optional，传入 null 返回一个空的 Optional

  • empty()：返回一个空的 Optional

  • isPresent()：判断值是否存在（最恶心的 API）

  • get()：如果此 Optional 存在值，则返回该值，否则抛出 NoSuchElementException（第二恶心的 API）

  • ifPresent(Consumer<? super T> consumer)：如果对象存在，执行 consumer

  • orElse(T t)：如果有值返回值，否则返回 t

  • orElseGet(Supplier<? extends T> other)：如果有值返回值，否则执行 other 返回值

    先比较一下执行结果

    • 从结果可以知道，orElse() 不论 Optional 容器中是否为 null，都会运行一下 createObject() 获取对象，但是 Optional 在容器为 null 的情况下不会执行供给型接口的方法

  • orElseThrow(Supplier<? extends X> exceptionSupplier)：值不存在抛出异常

  • filter(Predicate<? super T> predicate)：如果存在一个值，并且该值与给定 predicate 匹配，则返回描述该值的 Optional，否则返回一个空的 Optional

  • map(Function f)：如果有值，对其进行处理，并返回处理后的 Optional，否则，返回空 Optional

  • flatMap(Function<? super T, Optional<U>> mapper)：与 map () 类似，但是要求 mapper 返回的值必须是 Optional

新的日期 API

Java 原来的 Date API 是公认的设计很 “垃圾” 的 API，month 参数从 0 开始，year 默认加 1970，而且加减时间是在修改原来的值…… 新的日期时间 API 在 java.time 包下

LocalDate：包含年月日

LocalTime：包含时分秒

LocalDateTime：包含年月日时分秒

Instance：从 Unix 元年（1970 年 1 月 1 日 0 时 0 分 0 秒）到某个时间的时间戳

  1. 算时间差：

    1. Duration：计算两个 LocalTime 或者 LocalDateTime 的时间差

    2. Period：计算两个 LocalDate 之间的时长
