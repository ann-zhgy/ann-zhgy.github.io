linux 命令

¶ 常用命令

  1. 关机：shutdown -h，halt，init 0，poweroff

  2. 重启：shutdown -r，reboot，init 6

  1. pwd：查看当前工作目录

  2. ls：查看目录内容

    • -l：列表显示
    • -a：显示所有，包括隐藏文件
    • -h：人性化显示
    • -d：只显示目录，不查看内容

    目录结构：linux 的目录是从 ‘/’ 开始的，是根目录

  3. 注意事项：

    • 命令格式：命令 [选项] [参数]

    • 出问题查看帮助文档：

      1. 命令自带的帮助文档，一般是 -h 或 –help
      2. 找那个男人！man 命令

¶ 文件权限管理

  • linux 下，文件的权限有：所有者、所属组、其他人 的区分

    • ls -l 就能看到

  • 权限有：读 ®、写 (w)、执行 (x)

  • 使用 chmod 命令修改文件的权限

    • 格式：chmod [身份] [修改] [权限] 文件
      • 身份：u - 所有者；g - 所属组；o - 其他；a-all
      • 修改：+ 添加；- 去掉；= 设置
      • 权限：r (100) w (010) x (001) -(000)
    • 例如：
      • chmod u+w 1.txt ：为所有者添加写权限
      • chmod 754 1.txt ：设置权限为 所有者 rwx，所属组 wx，其他人 r

¶ 链接文件

  • 软链接：类似于 windows 下面的快捷方式
  • 硬链接：修改文件时会同步修改目标文件，注意，存在两个文件！！！
  • 命令： $ ln [-s] 源文件 文件名 # 创建链接到源文件的软链接，不加 -s 就是创建的硬链接

¶ 管道

  • | ：就是将前一个命令的输出作为后一个命令的输入
  • 实例：
    • ls -l /etc | more
    • ls /etc | grep '^vi'
  • 注意：有些命令是不接受管道参数的，可以使用 xargs 使用
    • find -type d | xargs rm -r

¶ 重定向

  • 打开任意程序，系统就会自动创建三个文件流：标准输入 (0)、标准输出 (1)、标准错误 (2)

    • 这三个文件流默认指向终端

  • 输出重定向：

    • ls /etc > 1.txt ：将 ls 输出的内容保存到 1.txt 中，文件不存在会创建，存在会覆盖
    • echo 'abc' >> 1.txt：将 ‘abc’ 输出到 1.txt 中，>> 表示追加

  • 错误重定向：

    • ls /abc 2> 2.txt：将错误输出重定向到 2.txt 中

  • 全部重定向：

    • ls /abc . &>3.txt：将所有输出重定向到 2.txt 中

¶ 系统服务

¶ 压缩解压

¶zip/unzip

  • zip zip-name file1 file2 file3...
  • unzip zip-name

¶gzip/gunzip

  • 压缩时不能压缩目录，会删除源文件。压缩文件后缀 gz

  • gzip 1.txt ：将 1.txt 压缩成 1.txt.gz

  • gunzip 1.txt.gz：将 1.txt.gz 解压

  • gzip -d 1.txt.gz：功能同 gunzip

¶bzip2/bunzip2

  • 用法同上，后缀 bz2
  • 说明：-k 参数可以在操作后保留原文件

¶ 打包解包工具：tar

  • 本身只提供打包解包功能，后缀是 .tar
    • 说明：tar.gz 简写为 tgz，tar.bz2 可以简写为 tbz2 或 tbz
  • 参数很多，1、2、3 不能同时使用
    1. -c：打包
    2. -x：拆包
    3. -t：查看内容，不拆包
    4. -f：指定文件
    5. -v：显示详细信息
    6. -z：使用 gzip/gunzip 工具
    7. -j：使用 bzip2/bunzip2 工具

shell 脚本

¶ 认识 shell

  1. shell 是介于用户和内核之间的一个工具用户的操作通过 shell 进行解析，然后传递给内核

  2. shell 有两种表现形式：交互式（单步执行），脚本式（文本执行）

  3. 我们通常说的 shell 编程时指第二种

  4. shell 类型

    1. bash（linux 默认）、ash、csh、ksh 等
    2. 环境变量 $SHELL 标明了系统使用的 shell 类型
    3. /etc/shells 定义了系统可用的 shell

¶Hello World

  • 添加执行权限：chmod u+x 文件名
  • 执行：
    1. 找到文件直接执行，需要添加执行权限，当前目录下： ./文件名
    2. 使用 shell 直接执行，无需添加执行权限：/bin/bash 文件名

¶ 规范

  1. 指定 shell 解析器：#! /bin/bash
  2. 注释符号：#
  3. 非常严格的使用习惯

¶ 变量

  1. 声明：name="123" 等号两边不能有空格

  2. 回收：unset name 回收后就取不到变量的值了

  3. 声明为只读：readonly name 声明为只读后变量的值就不能修改了

  4. 使用变量：

    1. 字符串内："${name} jdioafsfdlasd"

    2. 字符串外：$name

      ** 注意：** 加大括号的使用方式在哪里都不会有问题

¶ 变量类型

  1. 本地变量：只能在 shell 脚本内部使用
  2. 环境变量：在整个系统中都可以使用，通常全部大写，使用最多的是 PATH
    • 修改：
      1. 系统：/etc/profile
      2. 用户：~/.bash_profile
    • 导出：export PATH=$PATH:/root/shell
  3. 位置变量
    • $0：脚本文件的名字
    • $1-$9 ：接收的参数
  4. 特殊变量
    • $*：接受的所有参数
    • $#：参数的个数

¶ 文件包含

  1. . 文件名：包含当前目录的文件
  2. source 文件名：包含当前目录的文件

¶ 字符串

  1. 双引号：可以使用除 $, ", \, ` 外的所有字符

  2. 单引号：所有的字符都当作字符处理，不解析变量

  3. 反引号：表示命令执行的结果

    • echo `pwd` # /root/shell
